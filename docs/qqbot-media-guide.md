# 5 分钟拥有一个会发图、会说话的 AI QQ 机器人

文字聊天的 AI 就像一个只会打字的朋友——能聊，但总差点意思。

如果它能直接甩一张图给你解释代码、用语音给你念一首诗、把生成的报告以文件形式发到对话里呢？

**OpenClaw QQ Bot 插件**做到了。图片、语音、视频、文件，四种富媒体全部打通。更重要的是，从注册到跑起来，整个过程不超过 5 分钟。

---

## 第一步：注册 QQ 机器人（约 3 分钟）

前往 [QQ 开放平台](https://q.qq.com/) 注册账号（注意：不能直接用个人 QQ 登录，需要新注册）。

1. **注册账号** → 设置超级管理员（手机 QQ 扫码）→ 填写主体信息 → 人脸认证

*(截图位置：QQ 开放平台注册页面)*

2. **创建机器人** → 进入管理页面 → 拿到 **AppID** 和 **AppSecret**

*(截图位置：机器人管理页获取 AppID/Secret)*

3. **沙箱配置** → 开发管理 → 沙箱配置 → 单独聊天选"在消息列表配置" → 添加成员 → 扫码添加机器人

*(截图位置：沙箱配置页面)*

> 沙箱模式下机器人无需发布上架就能用，非常适合个人玩家和开发者。

---

## 第二步：安装并启动（约 2 分钟）

安装插件：

```bash
openclaw plugins install @sliverp/qqbot@latest
```

配置凭证：

```bash
openclaw channels add --channel qqbot --token "你的AppID:你的AppSecret"
```

启动：

```bash
openclaw gateway
```

三条命令，搞定。

**更懒的方式**——一键脚本：

```bash
./upgrade-and-run.sh --appid 你的AppID --secret 你的AppSecret
```

已经配置过？直接 `./upgrade-and-run.sh`，不带参数，用已有配置启动。脚本会自动处理旧进程、安装更新、启动网关，一步到位。

*(截图位置：终端启动成功的输出)*

## 开始玩：四种富媒体能力

启动成功后，你的 AI 机器人就不只会打字了。

### 发图片

AI 在回复中写 `<qqimg>` 标签，系统自动上传发送：

> **你**：画一只猫咪
>
> **AI**：好的，给你画了一只～ `<qqimg>/tmp/drawings/cat.png</qqimg>`

本地文件、网络 URL 都行，支持 jpg/png/gif/webp/bmp。

*(截图位置：机器人发送图片的聊天界面)*

### 发语音

配置 TTS 后，AI 可以把文字变成语音消息发出来：

```json
{
  "channels": {
    "qqbot": {
      "tts": { "provider": "openai", "model": "tts-1", "voice": "alloy" }
    }
  }
}
```

> **你**：用语音念一段《送别》
>
> **AI**：来听一段经典的《送别》🎵 `<qqvoice>/tmp/tts/songbie.mp3</qqvoice>`

支持 mp3/wav/silk/ogg 等十余种音频格式。没装 ffmpeg 也能用——系统会自动降级到 WASM 解码器。

*(截图位置：机器人发送语音的聊天界面)*

### 发视频

> **你**：发个演示视频看看
>
> **AI**：`<qqvideo>https://example.com/demo.mp4</qqvideo>`

支持本地文件和公网 URL，大文件（>5MB）会自动提示"正在上传..."。

*(截图位置：机器人发送视频的聊天界面)*

### 发文件

> **你**：把分析结果导出成 PDF
>
> **AI**：整理好了，请查收 `<qqfile>/tmp/report.pdf</qqfile>`

PDF、Excel、ZIP，什么格式都能发，最大 20MB。

*(截图位置：机器人发送文件的聊天界面)*

## 为什么体验这么丝滑？

不是因为功能多，而是因为**细节做对了**：

- **标签容错**：AI 写错标签名也没关系——`<qq_img>`、`<image>`、甚至中文尖括号 `＜qqimg＞` 都能被自动纠正，覆盖 30 多种变体写法
- **上传缓存**：相同文件短时间内发给同一目标，自动复用上次的上传结果，不重复传
- **有序发送**：一条消息里混着文字、图片、语音，系统自动拆解成队列按顺序发，某一项失败不影响其他
- **分层降级**：音频转换走 ffmpeg → WASM 解码 → 原始上传的多级 fallback，尽最大努力让语音发出去

好的产品不是功能堆得多，而是用户想用的时候——它就是好使。
